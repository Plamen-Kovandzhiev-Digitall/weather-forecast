Replace `src/weather-ui/src/App.tsx` entirely with the following content:

```tsx
import { useCityTemperatures } from './hooks/useCityTemperatures';
import { BulgariaMap } from './components/BulgariaMap';
// No App.css import — D-04 deleted App.css; all styles via Tailwind v4 utilities in index.css

export default function App() {
  const { data, loading, error, retry } = useCityTemperatures();

  return (
    // D-03: full-viewport shell, no header/footer chrome; map is the only persistent UI element
    // UI-SPEC Layout Contract: relative positioning context for absolute overlays
    <div className="relative w-screen h-screen overflow-hidden">

      {/* MAP: always rendered so tiles start loading immediately — visible beneath any overlay (D-11) */}
      <BulgariaMap cities={data} />

      {/* LOADING OVERLAY: absolute inset-0, z-[1000] — above all Leaflet layers (D-11, D-13) */}
      {loading && (
        <div className="absolute inset-0 z-[1000] bg-black/45 flex items-center justify-center">
          <div
            className="w-10 h-10 rounded-full border-4 border-white/20 border-t-white animate-spin"
            role="status"
            aria-label="Loading temperature data…"
          />
        </div>
      )}

      {/* ERROR OVERLAY: absolute inset-0, z-[1000] — above all Leaflet layers (D-12, D-13) */}
      {error && (
        <div className="absolute inset-0 z-[1000] bg-black/45 flex items-center justify-center p-4">
          <div className="bg-white rounded-2xl shadow-2xl p-6 max-w-sm w-full flex flex-col gap-4 border-l-4 border-red-500">
            <span className="text-3xl">⚠️</span>
            <h2 className="text-lg font-bold text-gray-900">Unable to load weather data</h2>
            <p className="text-sm text-gray-600 leading-relaxed">
              The weather service is unavailable. Make sure the backend is running at localhost:5055, then try again.
            </p>
            <button
              onClick={retry}
              className="bg-blue-600 hover:bg-blue-700 text-white text-sm font-bold px-4 py-2 rounded"
            >
              Try again
            </button>
          </div>
        </div>
      )}

    </div>
  );
}
```

Key implementation points:
- **`<BulgariaMap cities={data} />`** is always rendered — it accepts `null` (renders tiles only, no markers) and `CityTemperature[]` (renders tiles + markers). This ensures map tiles start loading immediately, even during the API fetch.
- **Loading overlay uses `{loading && ...}`** — conditionally rendered; the `animate-spin` class uses Tailwind v4's built-in keyframe animation.
- **Error overlay uses `{error && ...}`** — conditionally rendered; only shown when `loading` is false and `error` is set. The `retry` callback resets the hook's `retryCount`, triggering a re-fetch.
- **Both overlays use `absolute inset-0 z-[1000]`** — positioned over the map without affecting its layout (D-13). The z-index `1000` places them above Leaflet's markers (400) and controls (800).
- **Error copy is verbatim from UI-SPEC Copywriting Contract** — do not paraphrase or modify the heading, body, or button text.
- **DO NOT** import `App.css` (deleted in Plan 01 per D-04).
- **DO NOT** import `useWeatherData` (deleted), `WeatherMap` (deleted), or reference `VELIKO TARNOVO` (PITFALLS.md §14).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| useCityTemperatures error state → UI | Raw API error message stored in hook state; UI must display fixed copy, not the raw error |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Error overlay body text — could leak backend internals | mitigate | Error heading and body are **hardcoded fixed strings** in App.tsx JSX — the raw `err.message` from the hook is intentionally NOT rendered. Users see "The weather service is unavailable..." regardless of the actual error. Backend stack traces cannot surface in the UI. |
| T-04-02 | Denial of Service | Retry button triggers unlimited re-fetches | accept | Retry is user-initiated (one click = one fetch). No automatic retry loop. Backend has 10-minute cache (API-03), so rapid retries after success are cache-hits. Acceptable for a single-user dev tool. |
</threat_model>
