## Component Specifications

### 1. MapContainer (D-03, D-05, D-06)

> **Primary visual anchor:** temperature badge cluster over Bulgaria; the colour-coded badges are the first element the eye resolves against the neutral CartoDB Positron tile base.

```
Position:  fixed or html/body/root at 100vw × 100vh
Center:    [42.7, 25.5]  (Bulgaria centroid)
Zoom:      7  (immutable after mount — set correctly from the start)
Tiles:     CartoDB Positron
           URL: https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}{r}.png
           Attribution: © OpenStreetMap contributors © CARTO
scrollWheelZoom: true
className: "h-screen w-screen"  (Tailwind v4 utility; set on the wrapping div)
```

`html`, `body`, `#root` must all be `height: 100%; width: 100%; overflow: hidden`
(Tailwind: apply `h-full w-full overflow-hidden` to `#root` in `index.css`).

Leaflet attribution control: leave at default bottom-right position — **do not remove** (tile license requirement).

### 2. Temperature Badge (`L.divIcon`) (D-07, D-08, D-09)

```
L.divIcon({
  className: '',            // clears leaflet-div-icon default white box
  html: `<div class="{colorClass} text-white text-[13px] font-bold
                      px-2 py-1 rounded shadow-md whitespace-nowrap
                      leading-none select-none">
           {Math.round(temperatureC)}°C
         </div>`,
  iconSize:   [52, 28],    // bounding box for hit-test and centering
  iconAnchor: [26, 14],    // center of badge = exact city coordinate
})
```

`{colorClass}` is one of: `bg-blue-500`, `bg-teal-500`, `bg-green-500`, `bg-orange-500`, `bg-red-500`

**Memoization (required — see PITFALLS.md §6):** Wrap in a `CityMarker` sub-component.
Use `useMemo([tempC])` so the icon object is only recreated when `temperatureC` changes.

```tsx
// CityMarker component pattern
function CityMarker({ city }: { city: CityTemperature }) {
  const icon = useMemo(() => createTempIcon(Math.round(city.temperatureC)), [city.temperatureC]);
  return <Marker position={[city.latitude, city.longitude]} icon={icon}>...</Marker>;
}
```

### 3. City Name Tooltip (D-10)

```
<Tooltip direction="top" permanent={false} opacity={0.9}>
  {city.nameEn}          ← use nameEn field from CityTemperature type
</Tooltip>
```

Trigger: hover (Leaflet default for non-permanent tooltips).
Leaflet's built-in tooltip styles are acceptable — do not override tooltip CSS.
City name: English (`nameEn`) — readable for internationalised users.

### 4. Loading Spinner Overlay (D-11, D-13)

```
Position: absolute, inset-0, z-index: 1000 (above map tiles, below Leaflet controls)
Backdrop:  bg-black/45
Layout:    flex items-center justify-center

Spinner element:
  w-10 h-10
  rounded-full
  border-4
  border-white/20
  border-t-white
  animate-spin
  aria-label="Loading temperature data…"
  role="status"
```

Map tiles remain visible beneath the semi-transparent backdrop.
The spinner does **not** block map interaction (pointer-events: none on the backdrop, or accept
that brief interaction is blocked during the ~1-2s fetch — acceptable for this phase).

```
  2. Heading:        text-lg font-bold text-gray-900
  3. Body:           text-sm text-gray-600 leading-relaxed
  4. Retry button:   see §Copywriting
```

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Loading spinner aria-label | `"Loading temperature data…"` | Screen reader accessible; not visible |
| Loading spinner role | `"status"` | Announces to AT when overlay appears |
| Error card heading | `"Unable to load weather data"` | Plain language; describes the problem |
| Error card body | `"The weather service is unavailable. Make sure the backend is running at localhost:5055, then try again."` | Names the likely cause + explicit next step |
| Retry button label | `"Try again"` | Verb + context; no icon |
| Empty state | Not applicable — 28 cities always load or error; no partial empty state in v1 | — |
| Destructive actions | None in this phase | No delete/reset/irreversible actions |
| Map aria-label | `"Bulgaria weather map showing current temperatures for 28 district cities"` | Set on the `MapContainer` wrapper `<div>` |
| Tooltip content | `{city.nameEn}` — e.g. `"Sofia"`, `"Varna"`, `"Plovdiv"` | English city names from backend response |
| Badge text pattern | `{Math.round(temperatureC)}°C` — e.g. `19°C`, `−3°C`, `32°C` | No decimals (D-08); use `−` (U+2212) for negatives if formatting allows, else ASCII `-` |

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Map pan | Mouse drag / touch drag | Leaflet native pan — no custom handling |
| Map zoom | Scroll wheel / pinch / `+`/`−` controls | Leaflet native zoom — no custom handling |
| Marker hover | `mouseenter` on any badge | Leaflet `<Tooltip>` appears above badge with city name |
| Marker hover exit | `mouseleave` | Tooltip dismissed (Leaflet default) |
| Retry click | Click `"Try again"` button | Re-invoke the fetch hook; show loading overlay; clear error state |
| Map click (bare) | Click empty map area | No custom handler — Leaflet default (no popup) |
| Marker click | Click any temperature badge | No custom handler in v1 — no popup on click (popup details deferred to v2) |

## Data Flow Contract

```
App mounts
  → useTemperatureData hook fires fetch to http://localhost:5055/api/cities/temperatures
  → loading=true, data=null, error=null
      → render: MapContainer + LoadingOverlay
  
  On success:
  → loading=false, data=CityTemperature[28], error=null
      → render: MapContainer + 28 CityMarkers (no overlay)
  
  On failure:
  → loading=false, data=null, error="Unable to load weather data"
      → render: MapContainer (tiles still visible) + ErrorOverlay
  
  On retry click:
  → hook re-fetches → loading=true → success/failure as above
```

Type contract (D-15):
```typescript
interface CityTemperature {
  nameNative:   string;   // Bulgarian name e.g. "София"
  nameEn:       string;   // English name e.g. "Sofia"
  latitude:     number;
  longitude:    number;
  temperatureC: number;   // may be double? from backend — handle null/NaN gracefully
}
```

Badge render guard: if `temperatureC` is `null`, `undefined`, or `NaN`, render `"—"` instead of a number.

## Accessibility Checklist

| Requirement | Implementation |
|-------------|---------------|
| Map has accessible name | `aria-label` on `MapContainer` wrapper `<div>` |
