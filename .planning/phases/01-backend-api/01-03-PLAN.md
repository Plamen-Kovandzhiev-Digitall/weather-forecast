## Threat Model (ASVS L1)

| Threat | Severity | Mitigation |
|--------|----------|------------|
| HTTPS redirect loop: unguarded `UseHttpsRedirection` forces 307 on HTTP dev fetch, browser follows to non-existent HTTPS port | HIGH | Wrap with `if (!app.Environment.IsDevelopment())` — active in production, bypassed in dev (API-05, PITFALLS #4) |
| CORS bypass: all origins accepted in dev | MEDIUM | `AllowReactDev` policy restricts to exactly `http://localhost:5173` — not `SetIsOriginAllowed(_ => true)`. Keep the existing strict policy. |
| Information disclosure: unhandled exception details leaked in 500 response | MEDIUM | `CitiesController` catch block returns structured 503 `{ error, message }` with no stack trace or internal detail (D-02) |
| Mock data endpoint still live: `WeatherForecastController` returns random numbers that could mislead callers | HIGH | Delete `WeatherForecastController.cs` entirely (D-04) — confirmed by curl absence of `/weatherforecast` route |
| DI misconfiguration: missing service registration causes runtime `InvalidOperationException` on first request | HIGH | All four registrations in Program.cs: `AddHttpClient("open-meteo")`, `AddMemoryCache()`, `AddScoped<IOpenMeteoClient, OpenMeteoClient>()`, `AddScoped<ICityTemperatureService, CityTemperatureService>()` |

  Use C# 12 primary constructor syntax (dotnet-best-practices skill). Inject `ICityTemperatureService` and `ILogger<CitiesController>`.

  ```csharp
  using Microsoft.AspNetCore.Mvc;

  namespace WeatherForecast;

  /// <summary>
  /// Exposes GET /api/cities/temperatures — returns current temperatures for all 28 Bulgarian district cities.
  /// Delegates to ICityTemperatureService (which handles caching and Open-Meteo integration).
  /// Returns HTTP 503 with structured error body if Open-Meteo is unreachable after all retries.
  /// </summary>
  [ApiController]
  [Route("api/cities")]
  public class CitiesController(
      ICityTemperatureService service,
      ILogger<CitiesController> logger) : ControllerBase
  {
      /// <summary>
      /// Returns current temperatures for all 28 Bulgarian district cities.
      /// Results are cached for 10 minutes. On upstream failure returns HTTP 503.
      /// </summary>
      /// <param name="cancellationToken">Cancelled by ASP.NET Core if the browser navigates away.</param>
      [HttpGet("temperatures")]
      public async Task<IActionResult> GetTemperatures(CancellationToken cancellationToken)
      {
          try
          {
              var result = await service.GetAllAsync(cancellationToken).ConfigureAwait(false);
              return Ok(result);
          }
          catch (Exception ex)
          {
              // D-03: log on final failure
              logger.LogError(ex, "Failed to retrieve city temperatures after all retries.");
              // D-02: structured 503 body — gives frontend a machine-readable error signal
              return StatusCode(503, new
              {
                  error   = "upstream_failure",
                  message = "Could not retrieve temperature data from Open-Meteo. Please try again later.",
              });
          }
      }
  }
  ```

  **Why `[Route("api/cities")]` not `[Route("[controller]")]`:**  
  The `[controller]` token resolves to `Cities` (strips "Controller" suffix) giving `/Cities` not `/api/cities`. The CONTEXT.md explicitly requires the `/api/cities/temperatures` path — use the literal string.

  **Modify `src/WeatherForecast/Program.cs`** — two targeted changes, preserving all existing code:

  **Change 1: Add four new service registrations after `builder.Services.AddSwaggerGen();`**

  Insert the following block immediately after `builder.Services.AddSwaggerGen();`:

  ```csharp
  // Open-Meteo named HttpClient — PITFALLS #1 (IHttpClientFactory prevents socket exhaustion)
  // 10-second timeout — PITFALLS #9 (avoids 100s default hang on slow upstream)
  builder.Services.AddHttpClient("open-meteo", client =>
  {
      client.BaseAddress = new Uri("https://api.open-meteo.com");
      client.DefaultRequestHeaders.Add("Accept", "application/json");
      client.Timeout = TimeSpan.FromSeconds(10);
  });

  // In-process cache — API-03 (10-minute TTL on city temperatures)
  builder.Services.AddMemoryCache();

  // Service layer registrations
  builder.Services.AddScoped<IOpenMeteoClient, OpenMeteoClient>();
  builder.Services.AddScoped<ICityTemperatureService, CityTemperatureService>();
  ```

  **Change 2: Guard UseHttpsRedirection — API-05, PITFALLS #4**

  Replace the bare:
  ```csharp
  app.UseHttpsRedirection();
  ```

  With the guarded version:
  ```csharp
  if (!app.Environment.IsDevelopment())
      app.UseHttpsRedirection();
  ```

  **Do NOT change anything else in Program.cs:**
  - Keep `AddCors` with `AllowReactDev` policy for `http://localhost:5173` exactly as-is (API-06 ✓)
  - Keep `app.UseCors("AllowReactDev")` call
  - Keep Swagger dev-only block
  - Keep `app.UseAuthorization()` and `app.MapControllers()`

  **Why `TimeSpan.FromSeconds(10)` for HttpClient timeout:**  
  Default `HttpClient.Timeout` is 100 seconds. With retry logic, a slow Open-Meteo would hold a thread for up to 300 seconds (3 × 100s). A 10-second timeout per attempt caps total worst-case latency at ~30 seconds before 503 is returned (PITFALLS #9).
